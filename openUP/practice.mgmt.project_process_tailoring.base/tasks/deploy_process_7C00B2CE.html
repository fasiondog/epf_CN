任务：部署过程
该任务描述了如何向项目团队首次展示开发过程。
规程：环境
用途

此任务的目的：

  • 确保向项目成员介绍了过程。
  • 获取有关过程的任何反馈并在需要时完善过程
关系
角色主执行者:
  • 过程工程师
其他执行者:
输入必需:
  • 项目自定义过程
    		可选:
    • 无
    输出
    • 项目自定义过程
      步骤
      公开变更

      对任何重要的更新通知所有项目成员。

      培训团队成员

      除非变更非常微小，否则你需要对项目成员进行培训，介绍新的开发过程，包括指南、模板、工具。依赖于项目的规模和项目成员对类似开发过程的熟悉程度，培训方式可能非常不同，可以是从2小时的非正式介绍到更多时间的正式培训。

      以下是培训项目成员的常用方式：

      • 简报 如果变更很小或容易理解，一份简报可能就足够了。
      • 导师 项目拥有经验丰富的导师或顾问或对项目可用，是按需提供指导的一种有效方式。
      • 研讨会（Workshop） 所有项目成员参与的一整天的过程采纳会议研讨会（PAW: Process Adoption Workshop）有助于“激活”一个新的或重要变化的过程。
      • 定制培训课程 如果项目成员没有参加过过程和工具的标准培训课程，一种可选的方式是定制化标准的培训课程，覆盖项目的开发过程，包括指南、模板和工具。然而，它可能是昂贵的定制培训课程。一般过程培训，向过程或实践的介绍过程，应该在项目启动时优先引导，或者在项目早期的几天。更多特定的培训，如技巧、方法或技术，通常实时进行。这意味着培训是在方法或技术被应用到项目前不久，从而保证新的知识及时更新。
      • 夏令营"Boot-camps” 1~5周的集中实习培训，没有许多组织能够负担得其这种引导夏令营式的培训，但当项目的人员中存在很大新的因素时，这被证明是一种有效的方式。这种夏令营方式通常混合了研讨、培训课程以及过程和工具实习。
