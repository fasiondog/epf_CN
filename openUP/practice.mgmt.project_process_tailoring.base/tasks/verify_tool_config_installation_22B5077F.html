任务：验证工具配置与安装
此任务描述了如何验证项目的开发配置已经可用。
规程：环境
用途

此任务的目的是验证开发系统需要的工具已经可用。

关系
角色主执行者:
  • 工具支持专员
其他执行者:
输入必需:
  • 工具
    		可选:
    • 无
    输出
    • 工具
      主要描述

      完成对工具和开发基础设施的验证，在项目开始使用它们之前。很显然，如何完成此项任务依赖于不同的技能、技术以及工具。

      步骤
      验证环境

      验证环境，包括正确的硬件、软件和数据。验证正确的硬件被安装。这可能可以直接观察到或者通过工具（如使用“我的电脑”右键中的属性）。

      验证工具

      验证安装了正确的软件配置。这可以通过查看注册表、ini文件或者工具本身自带的配置信息或选项来进行。

      验证数据

      验证包含了合适的数据。验证数据可能需要使用数据可视化工具，或使用某些应用程序显示这些数据。在某些点，可能需要在每个工具上选择和执行一个或多个场景，以确保使用工具的结果和预期的一致。

      运行工具

      召集一个了解 工具 和 项目自定义过程 的小团队，并让他们运行工具。

      • 测试多站点，更多的冲突用户
      • 至少对每个工具执行一个用例场景，来验证对工具进行了合适的安装和配置。
      • 试验一下在工具指南中的那些开发过程的正常场景。
      • 测试和其他工具的集成。

      需要的话，提交问题变更请求。
