工件：支持文档
供产品支持团队使用的文档，提供了有关如何服务并支持指定产品的相关信息。
域：部署
用途

此工作产品的目的是为产品的不同组织层次（层1、2、3）提供足够的信息，以便有效的为应用提供支持。

关系
角色负责人:
  • 技术作者
修改者:
  • 技术作者
任务输入至:
  • 提供支持培训

  • Develop Training Materials
输出自:
  • Develop Support Documentation
描述
主要描述

支持文档通常是为了支持组织的三层通用结构而开发的。层1通常是帮助台（Help Desk），当用户遇到问题时可以询问。层1支持人员通常回答基础问题，如果需要，记录问题并将其提交到合适的层1支持台。

层2支持人员可以处理更复杂的应用问题，并且可能需要研究一些系统特征来提供答案。如果此层支持人员无法解决该问题，需向层3支持人员提交问题，层3支持人员对应用代码和支持系统架构的技术有深入的理解。

为了向每个支持层正确的传递必要信息，应用代码应该被良好注释并被有逻辑的组织。这种方式将促进支持文档的开发。

支持文档通常包括：

  • 用户手册，包括用户使用说明、使用过程描述
  • 沟通、培训以及可传递知识的交付成果
  • 支持和操作手册
  • 服务信息，包括帮助台脚本
