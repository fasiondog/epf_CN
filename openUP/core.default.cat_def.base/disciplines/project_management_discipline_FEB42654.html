规程：项目管理
此规程解释了如何指导、促进及支持团队，帮助他们处理构建软件过程中出现的风险和障碍。
关系
任务
  • 评估结果
  • 管理迭代
  • 策划迭代
  • 项目策划
  • 变更请求
主要描述

此规程的目的：

  • 鼓励利益相关人对工作顺序的优先级达成共识
  • 鼓励团队为项目创建长期和短期的计划
  • 聚焦于团队为利益相关人评估持续的交付经过测试的软件
  • 帮助创建高效的工作环境以最大化团队生产率
  • 保持利益相关人和团队对项目进展的了解
  • 提供一个管理项目风险和持续适应变化的框架

项目管理规程是一个保护性的规程，影响或被影响了所有其它的规程。项目管理活动为创建高性能的工作环境创建了价值：

  • 利益相关人信任团队成功交付价值的能力，并理解技术平台的能力和权衡。
  • 项目团队成员理解利益相关人意图，并通过持续的生产可工作的软件产品进行评估来确认这种理解。
