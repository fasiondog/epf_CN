规程：开发
此规程解释了如何设计和实现符合架构并支持需求的技术解决方案。
关系
任务
  • Implement Developer Tests
  • Implement Solution
  • Run Developer Tests
  • Integrate and Create Build
  • Design the Solution
主要描述

此规程的目的：

  • 将需求转化为系统设计
  • 适应设计以符合实现的环境
  • 增量的构建系统
  • 验证构建系统的技术单元符合规格

通过每次的迭代，此规程中的任务将演进一个越来越富有能力和稳定的构建（Build）。

当在系统上工作时，开发人员将使用并受到架构的约束。

此规程通过以下途径和其它规程关联：

  • 需求 规程定义了将设计和实现什么。
  • 架构 规程组织并约束了设计和实现
  • 测试 确认系统构建符合需求
  • 项目管理 规程计划了每个迭代中将实现的功能，并提供了管理系统变更的机制。
