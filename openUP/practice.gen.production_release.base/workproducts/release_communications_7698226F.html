工件：发布通告
此工件通知各相关方产品已经部署到生产环境。
域：部署
用途

此工作产品的目的是通知所有不同利益相关人，发布已经部署到生产环境，且已实现的特性已经可用。

关系
角色负责人:
  • 部署工程师
修改者:
  • 部署工程师
任务输入至:
  • 交付发布通告
输出自:
  • 编写发布通告
描述
主要描述

有时，依赖于产品用户基础，可能需要为每个利益相关人群体准备单独的通知。在此种情况下，此工件应该为不同群体指定不同的通知，通知的方法（例如，email、文本或纸质通知、公告、通讯、电话等等）。所有的通告应该事先准备，这是传播信息的态度，当发布投入生产是已决定一定会成功。

此工件中除通告的时间和依赖关系外，还包含一份发布成功时负责执行通知的责任方（通知是部署工程师）列表。

虽然没有规定发布通告的格式，但是每个通告应该指明优先的交付机制（例如：通知、电话、网站上的内部公告、通过高级管理人员在线或预制的演示，等等）。此工件通常回答了如下问题：

  • 当发布投入生产时，有哪些感兴趣的相关方？
  • 有哪些（特性、功能、组件）投入了生产？
  • 为什么此发布对利益相关人有价值，以及它的目的是提供什么服务？
  • 产品在何处（什么平台、地理位置、业务单元，等等）？
  • 利益相关人如何访问系统以及需要在什么环境下？
  • 产品何时发布（或者如果是在未来日期发布，那么具体是何时）？
