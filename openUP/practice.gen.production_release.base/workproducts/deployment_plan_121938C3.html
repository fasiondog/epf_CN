工件：部署计划
部署计划用于记录部署工程师成功部署一个版本需要的所有信息。
域：部署
用途

此工件的目的是在一个文档中描述部署工程师在部署某个发布包之前和过程中需要的独特信息。

关系
角色负责人:
  • 部署工程师
修改者:
  • 部署工程师
任务输入至:
  • Develop Release Communications
  • Execute Deployment Plan
  • Install and Validate Infrastructure
  • Package the Release

  • Develop Backout Plan
  • Verify Successful Deployment
输出自:
  • 策划部署
描述
主要描述

部署计划应该包含部署产品特定版本的独特指令。“独特指令”是指不包括那些部署工程师正常程序中的内容。部署计划包含的是特定的过程和时间限制，部署工程师应该意识到他们正在部署某个特定的版本。

通常，部署计划的草稿版本由开发团队制定，而部署工程师负责其中的内容。部署计划通常包含以下章节：

  • 版本范围以及部署的功能概述
  • 将组件部署到不同节点的时机和依赖关系
  • 版本相关的风险或问题
  • 版本将影响的客户组织、利益相关人、最终用户社区
  • “准备生产”的批准人
  • 将版本包交付和部署经理的开发人员及其联系信息
  • 将版本包转交给部署工程师的方式，包括适当的联络方式及程序
  • 部署成功的关键准则；换句话说，部署工程师如何知道并报告该版本已经被成功的部署。
