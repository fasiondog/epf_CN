核对表：测试脚本
此检查单提供了一系列问题来验证以一致并完整的方式创建了测试。
关系
相关元素
  • 测试脚本
检查项
测试脚本符合相关测试的用例

确保测试脚本符合相关测试用例规约。测试用例表达了测试的目的，测试脚本必须符合这些目的。

测试脚本是可测试的
测试脚本是可重用的

通过最大化可重用设计来保证测试脚本能够被重用。促使重用有不同的方式，这依赖于是生成、编程还是记录测试脚本。

测试脚本是规范且明确的

确保测试脚本表达了清晰的指令，这些指令指示了测试必须如何运行以及应该如何分析结果。虽然在编写非自动测试时，测试人员能够在测试如何精确运行上有所保留，但在对测试结果（成功或失败）的分析上并没有创新的余地。

测试脚本的命名与其它测试工作产品一致

确保你的测试脚本的命名与其它相关测试工作产品一致。例如，如果你为每一个测试用例创建了测试类，那么确保它们的名称表达这种关系。此外，如果你正在将测试脚本构建在某个库中，那么使用一致的命名惯例来反映这些库。

测试脚本提供了合适的测试覆盖率

确保你的测试脚本提供了和被测系统一致的测试覆盖率。
