工件：测试脚本
此工件包含了组成测试并使其可运行的一步一步的指令。文本脚本可以采用的形式包括：手工记录的文本指令，或者自动测试所用的计算机可读的指令。
域：测试
用途

测试脚本以一种有效率和效果的方式实现了测试所需的某个子集。

定制
不具有的影响

没有此工件，将难以保证测试被一致的运行，通过一致的运行可重复结果并可重现错误。如果没有使用测试脚本，将难以验证错误的出现是否是因为运行测试的方式本身引起的。

不需要的原因

如果测试很简单或已经在某种非正式的基础上做过测试，那么可能不需要此工件。然后，为了保证这种方式是成功的，测试人员必须跟踪他们的处理，因为他们可能需要重现场景中的问题。或许，你可以使用一个自动捕捉和回放的工具来提供测试的记录。
