工件：测试日志
此工件收集了在单独的测试周期中执行测试所产生的原始输出。
域：测试
用途
  • 为执行的一系列测试提供验证
  • 为那些成功的测试提供相关信息
关系
完成的槽
  • [技术-测试结果]
角色负责人:
  • 测试人员
修改者:
  • 测试人员
  • 开发人员
任务输入至:
  • 评估结果
输出自:
  • 运行测试
  • 运行开发人员测试
描述
主要描述

此工件提供了详细的，通常基于时间的记录，这些记录验证了执行的一系列测试，并为这些成功执行的测试提供了相关信息。重点通常在于提供准确的审计跟踪，能够使你在运行后对失败进行诊断。随后分析这些原始数据，来确定测试工作某方面的结果。

定制
不具有的影响

没有此工件或类似的文件，将没有运行测试的记录，也就无法发现差异以及确定需要采取的行动。如果此信息无效：

  • 无法知道哪些测试执行成功、哪些失败
  • 无法了解测试的状态，以及测试对应层面的产品质量
  • 无法了解还剩余哪些测试
  • 可能存在合同和法律上的问题
不需要的原因

当执行自动化测试时，将自动产生测试日志。通常，问题不在于是否产生测试日志，而在与是否保留记录，以及在哪里保留记录。对于手工测试，问题在于是否保留单独的测试日志或以另一种形式保留测试结果的总结。

说明选项

因为这是为了后续的分析收集的原始数据，它可以有多种表达方式：

  • 对于手工测试，在手工测试脚本中记录实际结果
  • 对自动化测试，直接将输出保存在日志文件中从而可以追溯到自动化测试脚本
  • 在测试管理工具中跟踪原始结果数据
