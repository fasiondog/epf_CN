工件：用例模型
此工件描述了关于系统预期功能和环境的模型，该模型可作为客户和团队之间的合同。
域：需求
用途

此工件表述了系统预期功能的概貌。是利益相关人和项目团队之间就系统预期功能达成一致意见的基础。同时也指引了软件开发生命周期中的不同任务。

定制
不具有的影响

没有该工件，将很难判断参与人与用例之间的关联关系，同时也难以理解不同用例之间的关联。

不需要的原因

你可以不需要此模型，如果所有的用例集合非常小且不存在复杂的关联关系。

如果使用了其它的方式文档化需求，也可以不使用用例模型。

说明选项

表达的方式包括：从UML建模工具产生的报告和图表、其它图形绘制工具绘制的图形、在白板上进行绘制。更多的用例模型信息在用例规约中表达。
