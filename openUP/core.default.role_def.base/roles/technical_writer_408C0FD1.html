角色：技术作者
技术作者（Technical Writer)经常帮助开发团队成员完成产品负责人、终端用户、支持人员所需的文档，使他们理解并能够使用交付的特性。
角色集：部署实施
关系
修改
  • 产品文档
  • 用户文档
  • 支持文档
    主要描述

    某些组织可能雇佣技术作者来帮助开发团队成员Review并完善文档，这些为交付特性提供支持的文档由团队在Sprint/迭代中输出。

    由于技术作者经常善于编写并维护技术文档，他们有助于保证文档的一致性并符合企业标准。

    人员配备
    技能

    技术作者应该具备以下技能：

    • 优秀的技术写作能力
    • 理解复杂技术概念并将它们转化为易于用户理解的能力
    • 编写简洁的帮助文档并使其成为应用程序一部分的能力
    • 从开发人员、测试人员以及产品负责人那里获取技术细节，并以易于最终用户和支持人员理解的方式记录这些细节的能力
    • 坚持企业文档标准的能力，如果不存在标准，那么应该有能力编写这些标准并和开发团队沟通
    • 对正在开发的特性所在领域有深刻的理解
