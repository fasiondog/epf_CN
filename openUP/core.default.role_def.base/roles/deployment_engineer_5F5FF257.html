角色：部署工程师
部署工程师（Deployment Engineer）负责将一个或多个发布版本安全的部署到生产环境。
角色集：部署实施
关系
其他执行
  • Package the Release
  • 制定回退计划
修改
  • 部署计划
  • 发布通知
  • 基础设施
    主要描述

    部署工程师协作部署经理。此团队角色对生产环境的安全至关重要，帮助阻止在生产环境中引入坏的或未经测试的代码。部署工程师通过完成任务来支持部署经理，持续的领导、促使并协调同步的版本，通过使用流程程序来最大化交付给客户的价值。

    人员配备
    技能

    部署工程师需要具备：

    • 交叉平台经验以及识别并解决集成问题的经验
    • 系统级和集成级别的测试知识
    • 强有力的质量保证背景
    • 对持续集成实践的优秀理解
    • 精通特定的和实现的测试，以及生产环境基础设施
    • 自动化测试产品的专业知识，包括如何设置自动化测试、如何编写自动化测试脚本、如何创建自动化回归测试套件，以及如何解决自动化测试的问题
    • 理解发布和部署管理过程，了解ITIL（Technology Infrastructure Library ）基础
    • 能够和团体中的其他人员一起良好工作
