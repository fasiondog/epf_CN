角色：部署经理
这个角色（Deployment Manager）负责在一个或更多的项目和/或产品线中管理和调度同步发布的版本。
角色集：部署实施
关系
主要描述

此角色也被称为“部署协调员”或“部署促进者”，因为其主要职责是协调和促进开发的产品化软件交付，该交付通过某个程序交付给市场部门以作为通用可用性（general availability ：GA），或IT运维部门以将其部署到生产环境中。

部署经理的其它职责包括：

  • 管理支持团队的大部分日常工作
  • 在每次发布中，辅助程序经理以及开发团队成员制定计划
  • 确保文档化组织的发布控制，并使其被开发团队和程序支持团队良好理解
  • 确保应用程序的架构和指令可以被健壮并稳定的部署
  • 确保文档化了详细的部署计划，且该部署计划中包含了部署出现错误时的回退计划
  • 协调市场部门和程序，确保已经准备好并Review了所有利益相关人之间有约束力的沟通。
  • 确保产品通过程序被正确且完成的进行了集成
  • 在部署之前，验证正确的打包了产品，并确认其已经满足了所有发布控制
  • 同市场部门或IT运维部门一起成功的部署产品
  • 向所有利益相关人发布有关产品的事先计划的沟通
  • 引导发布回顾，同所有在程序中参与的开发团队一起，改进发布过程以及提高程序生产率和产品质量
人员配备
技能

此角色应拥有如下能力：

  • 良好的技术领导能力
  • 对产品发布方法论有优秀的理解（包括熟悉ITIL的发布和部署管理过程）
  • 丰富的系统集成经验
  • 丰富的策划复杂软件产品部署和交付的经验
  • 对产品架构和组件指令的良好理解
  • 对市场功能的良好理解
  • 政治才能
