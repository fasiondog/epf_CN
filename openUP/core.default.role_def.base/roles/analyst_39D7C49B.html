角色：分析人员
该角色代表了客户和终端用户的关注点，通过从利益相关人处收集输入，以理解待解决的问题，并捕获需求和设置需求优先级。

    人员配备
    技能

    分析人员需要以下的知识、技能和能力：

    • 识别和理解问题与机会的专业技能
    • 具备良好的表达能力，表述待解决的关键问题或待实现的机遇。
    • 具备和扩展团队高效协作的能力，通过合作工作会议、务虚会（workshop），联合应用开发会议（JAD sessions: joint application design）以及其他技术
    • 良好的沟通技能，口头及写作技能
    • 业务和技术领域的知识或者快速吸收和理解此类信息的能力
    分配方法

    通过以下方式分配此角色：

    • 由团队中的的一人或多人专职承担。这一普遍采用的方式适合于存在难以收集的复杂需求的情况。
    • 由团队中的一人或多人同时承担此角色和测试软件的工作。这对较小的测试团队或资源受限的测试团队来说，是个不错的选择。
    • 由客户代表承担此角色。此方法适合于敏捷团队，在敏捷团队中客户是团队的一部分并且可用。

    • 参见指南：Staffing a Project，了解在小的敏捷团队中如何分享该角色

    • 参见指南：Self-Organize Work Assignments，了解在小的敏捷团队中如何分享该角色
